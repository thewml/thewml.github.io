UTF-8 support

  • From: Denis Barbier <nospam@thanx>
  • Date: Thu, 19 Jul 2001 17:56:39 +0200
Hi,

in the past, some people requested the ability to change input encoding.
It seems quite hard to make WML deal with encodings which are not
ASCII-compatable, and I plan to only support ASCII-compatable 8bit
encodings (which are already supported through locales) and UTF-8.
Of course, I won't write any UTF-8 huge library, but use new Perl
(for passes 1, 3, 5, 6, 7, 8 and 9) and PCRE (pass 2) features when
detected at configure time. So all passes except wml_p4_gm4 should have
this support soon, which will be improved by Perl and PCRE future
versions.

Any comment or experience on this issue ?
