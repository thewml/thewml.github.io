> To make it possible to switch from one language to the other at any

I had the same problem for my pages in Esperanto. This artificial
language uses some non-US-ASCII characters, and with Unicode support
still missing on many systems, I have Unicode, ISO-Latin-3, and a
couple of common transliterations. Basically I have a file where I put
some general information on the different pages of a given site, and
this file is read into the templates for the others. And then, the
true files have names that differ only in a predictable manner -
e.g. appending some (language) code. A simple perl loop can then
create what you need. If you want, I can send you some wml files that
do just that (probably it would not be a good idea to post them on the
list, I guess), but they don't use navbars - I'm a lynx user.
