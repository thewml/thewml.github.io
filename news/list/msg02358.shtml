I'm getting close to the reason!

You're using <!DOCTYPE html PUBLIC "-//W3C//DTD HTML 4.0 Transitional//EN">,
I'm using <!DOCTYPE html PUBLIC "-//W3C//DTD HTML 4.01//EN">

If I switch to a "transitional" doctype too, I have no problems
either :-)

I've been using "tidy" with --doctype strict, with causes tidy
to use my regular doctype entry. Using --doctype loose 
results in the "transitional" doctype.

I'll still fiddle with the vertical alignment, to see
if NS 6.1 really needs the "traditional".
