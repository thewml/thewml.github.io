I've looked after the problem of generation of the script handling the
onMouseOver (tag rollover)
Here is the problem :
when the first rollover tag is in a slice, the javascript handling the
onMouseOver is generated in that slice
if another rollover tag is encountered in another slice, the script is NOT
generated in that slice
Depending on the configuration of the slices, and especially if they are
independant, the html generated for the second
slice doesn't have any script for the onMouseOver, thus generating an
javascript error when viewing the page
Does a patch exist to handle this problem ?
In fact, a work around is to define a rollover outside all slices, at the
beginning of the file, but ...........
