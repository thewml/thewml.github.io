"tidy", doctype, and "summary" attribute in tables

  • From: Stephan Petersen <nospam@thanx>
  • Date: Thu, 3 Aug 2000 14:51:02 +0200 (MEST)

Hi everybody,

"tidy" likes to complain about missing "summary" attributes in
<table> tags.

While I can of course easily add them for my own tables, 
wml tags like <tul> and <grid> are a bit trickier.

Would it make sense for wml to provide summary="" as default for tables
created by these tags?

Btw, the reason why I stumbled over this is that tidy doesn't think
a doctype like

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 3.2//EN">

is valid for files created by wml (I'm not quite sure why, since it
doesn't complain about *all* files. It might have something to do with
Javascript functions). It considers the wml output to be more of a
doctype like

<!DOCTYPE html PUBLIC "-//W3C//DTD HTML 4.01//EN">

The only thing it complains about in this case is the "summary"
attribute for tables...


All the best,

Stephan
