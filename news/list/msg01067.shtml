BUG REPORT

Package: WML 1.7.4
Operating System: AverageOS 0.8.15

Problem Description:
| (feature request)
| 
| WML_SRC_ISOTIME and other time variables defined
| by wml(1) provide the time string adjusted to the
| current user's timezone settings.  In our environment,
| fixing the timezone to GMT would be much more
| appropriate.  Though we could require each user
| to adjust their TZ settings before running wml(1),
| this is ackward.
| 
| I suggest that some mechanism to specify the
| timezone to use for these variables be provided
| OR that GMT versions of the defines be provided.
