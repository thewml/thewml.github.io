At what point does this sort of thing cross that boundary of "this is
XML's job?"

It would seem to me to be better to develop a formal orgchart dtd and
develop a renderer for it, and plug that into wml (define a tag which
expands into the necessary perl code to system() or exec() or whatever...)

Don't get me wrong, there is something to be said about the
lightweightedness of wml, and I surely don't think we should impose strict
xml semantics on navbar, etc, but there does have to be some line, surely,
where we cross it and say "alright, this is so involved that it's best to
do it more formally with xml."
