On Sun, 19 Mar 2000, Wolfgang Dautermann wrote:

> Hi,
> 
> today I found, that wml::std::lang provides (for example) either
> <de: >, <en: > Tags (if one uses the option "short") or
> <lang:de> </lang:de>,  <lang:en> </lang:en> Tags 
> - but never both variants. Is this intentional or a bug?
> If it is intentional one should clarify that in the documentation, but
> I think it would be better if the "short" Variant would provide the
> "long" tags in addition - so that one can switch to the "short"
> variant without having to change all the "long" tags.
[...]

Hmmm, i believe that documentation is clear enough.
By default
    <lang:en: >
    <lang:en> ... </lang:en>
With the `short' attribute
    <en: >
    <en> ... </en>

Anyway i see no reason not to always define the long version, and will
modify wml::std::lang.
