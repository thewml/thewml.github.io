Denis Barbier <barbier@imacs.polytechnique.fr> writes:

> It does not really matter, but -o(ALL-LANG_*)+LANG_EN is easier to
> understand. It means all slices minus the ones relative to linguas
> plus LANG_EN. Even not technical people will find this simple :o)

The problem is that we are talking about REALLY non-technical people.

One reason of using WML in our environment is that adding and
modifying context can be nicely separated from the HTML design.

Ideally that would mean that the "editors" have to only learn very few
<tags> and everything else works automatically.

To make nav-bars work one already needs some "complicated" #include or
#use statement at the beginning of an input file. 

The shebang line adds just one more thing to it.

> Moreover, you can replace ``index'' by ``%BASE'' as explained in wml(1).

That was one feature we were looking for, great that it's there!!!
 
> But it is only available on the shebang line, so i see no solution
> to your problem (except with a makefile).

I almost think we might indeed want some make based approach that
would ultimately also deal with the nav-bar issues. I guess we have to
look into that. With the right preprocessor one could just put files
into a directory structure and have the preprocessor do all the magic
with nav-bars, etc.

> One could write -o(ALL-LANG_*)+LANG_IT:%BASE.it.html in the .wmlrc
> files, this feature will go in my to-do list. 

Can you put it at the top   ;-)
