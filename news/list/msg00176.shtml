Style Sheets and WML support?

  • From: "Ralf S. Engelschall" <nospam@thanx>
  • Date: Sat, 1 Aug 1998 19:45:17 +0200

I've recently created a few webareas with Style Sheets (actually CSS1) and
recognized some WML flaws: I'd to prefix own definitions `#foo { ... }' with
two escaping backslashes, etc. This is ugly and already fixed for WML 1.6.7
now, of course. 

But I'm now thinging about Style Sheet support and asking myself: Can we do
something to make Style Sheets easier to use within WML by providing some
tags, some filters, whatever. Has anybody deeper experiences on working with
Style Sheets and WML and already some suggestions on how to make life easier?

What still missing functionality (which can be implemented in realistic amount
of time, of course ;-) would be cool to have inside WML when working with
Style Sheets?
