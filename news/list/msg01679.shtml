Bug in wml::std::lang

  • From: Corsin Decurtins <nospam@thanx>
  • Date: Wed, 31 May 2000 10:13:31 +0200
I'm using wml-2.0.1 and I have a problem with wml::std::lang.

Here's an example:

  #use wml::std::lang

  <lang:new id="de" short>
  <lang:new id="en" short>

  <lang:star:slice: index.*.html>

  <a href="<lang:star: index.*.html>">Link</a>

If I try to run wml, I get this error message:

  Bad regular expression `*': Invalid preceding regular expression

There is no error, if the wildcard is set to "%":

  <lang:set-wildcard "%">
