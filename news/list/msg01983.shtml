Hello, 

i decided again to solve my problem with current wml by patching
navbar.wml for the following:

a) as there are alt, hint and txt parameters i see it naturally to
create navbars with both text and images in the anchor tag. My "patch"
only appends $txt at the place where anchor output occurs. My patch
breaks only navbars where one uses txt parameters when alt or hint are
meant.
 
b) the patch also gives the anchor a stylesheet class $name.

The not so intrusive solution to write a navbar filter function would
have been really ugly and had to be repeated in all my navbars.
