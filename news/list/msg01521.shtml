On Sun, 9 Apr 2000, Tobias Zimpel wrote:

[...]
> wmk should not skip .wml-files using #influde=file if file has changed.
[...]

IIRC wmk skip system files but not user files, e.g.
#include <foo/bar.wml>
   is skipped but not
#include "foo/bar.wml"

The reason is that system files are presumably immutable (and it saves
time).
