Plans for WML 2.0.3

  • From: Denis Barbier <nospam@thanx>
  • Date: Tue, 1 Aug 2000 10:17:26 +0200
Hi,

i will be away next week for 3 weeks, without net connectivity.
During my vacation, i plan reading books on Perl regular expressions
(especially how to reduce regexp processing time) and writing efficient
C programs. Tell me if you know good books (or eventually URL pointers)
on these topics (even classical ones). Please do not reply to this
mailing-list when sending your advice.

WML 2.0.3 will be released at my coming back, after some final checks.
If some of you could test current snapshot and report whether it works
or not, it would help me a lot.  It contains no known bugs, works well
with my input files, but almost all modules have been updated, so some
bugs may have been introduced.

Since my last report about wml-SNAP status, following changes have been
made:
* Added attribute on <subst-in-string>, <subst-in-var> and <match>
  to use `imsx' perl modifiers.
* All modules have been updated to allow XHTML compliant output.
* Passes 7, 8 and 9 are soooo slow on large files. they now run up
  to 10 times faster.
* <protect> tags are now nestable.

See http://www.engelschall.com/sw/wml/distrib/wml-SNAP/ChangeLog
for a more detailed list.

It would be nice to have time comparisons with WML-2.0.2, some fixed
bugs may increase processing time.
