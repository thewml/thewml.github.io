I guess it is the difference between a development version (1.9.3) and the
stable version (1.020) at least that is what Neil says on
http://www.weblint.org. I use debian unstable myself, but I prefer to
see stable versions of toolkits like wml to ship with stable versions of
3rd party software (if you can call it this way in this case). I guess the
unstable version of wml could (should?) have the unstable version of
weblint included. 1.99.4 is supposed to become wml 2 which is supposed to
be stable (I hope I got that right) so I guess no unstable software should
be put into these packages.
