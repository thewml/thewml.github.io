On Mon, Nov 29, 1999, Stephan Petersen wrote:

> the latest version of gd (http://www.boutell.com/gd/) doesn't support the
> creation of GIFs any more. Does/should that have any effect on wml/gFont?

Yes, T.Boutell switched to PNG because the GIF license issue.  But it doesn't
directly affect gFONT, because currently gFONT still uses it's own local gd
library and this was patched to support the loading of the TeX/gdf fonts, etc.
What actually should be done is to perhaps move from this old gd library to
the recent one, but only if GIFs can be still generated for backward
compatibility.
