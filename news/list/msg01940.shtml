Yes! Please avoid such "dirty" tricks like adding unwanted
options to tags per se. It may (or may not) break standards.

Please allow that I add some "critisism" towards wml 2.0.x: I
strongly suggest a split-up in a developers line and a
stable production line like it is been done with linux.

We still keep our hands away from wml 2.0.x since I read too
many times about bugs and problems such as the things above.

If you like to see wml as a professional web-meta-language
you have to make sure, that it is consistent in its use.

I suggest that you skip wml 2.0. name it 2.1 and release
the really stable and tested(!) version as 2.2.
Please consider that testing nowadays uses around a 3rd of
the time of the whole progress. You cannot almost skip it and
do it like Microsoft (or Intel lately) and have the enduser
test the results. It bothers the producer as well because
of all the patches he has to take care of instead of
developing further.

Still wml is _great_ software and I don't want to miss it.
I use it every day.

Just my two cents
 -jan
