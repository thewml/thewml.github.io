Hi Denis,

I have a (hopefully) small request for the next release:

Could we have WML_SRC_TIME in addition to WML_SRC_CTIME? This would
make it a lot easier to customize the output format (with strftime).

Right now I am first parsing WML_SRC_TIME into
($sec,$min,$hours,$mday,$mon,$year), run this through timelocal() and
then use strftime.

Cheers, 
Fritz
