The Debian web site currently uses "make install" to copy all the *.html
files etc. to a different directory.  Two reasons that I can think of are:

  1. The public can't view the .wml files and Makefile etc. directly.
     (safer for Debian.)  :-)

  2. Less stuff for worldwide Debian mirrors site to mirror.

There may be other reasons that I have missed.  I wasn't the one who set
up all the supporting Makefile and templates for the Debian web site.  :-)
