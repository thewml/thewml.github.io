On Wed, 8 Sep 1999, Fritz Zaucker wrote:

> PROBLEM DESCRIPTION:
> 
> <url>At our web site (http://www.ee.ethz.ch/) ...</url>
> 
> gets translated into 
> 
> At our web site (<a
> href="http://www.ee.ethz.ch/)">http://www.ee.ethz.ch/)</a> ...
> 
> NOTE: The URL includes the closing parenthesis )

You're right.

After reading the RFC1738, i think that parenthesis are valid characters
within URLs. My first thought was that i had to respect this fact.

But now i believe that <url> is an help tag, and so is not intended
to be fully RFC compliant. I will remove the last character of the URL
if one of ['"?!.;:)>] (brackets are not included in this list).
Any comment?
